## Et si on arrêtait de se prendre la tête avec la documentation ?

Guillaume Scheibel présente **Asciidoctor**, la solution pour réconcilier les développeurs avec la documentation.

### Le problème
Écrire de la documentation dans un fichier complètement distinct du code crée une frustration permanente : elle devient rapidement obsolète, personne ne la maintient, personne ne la lit.

### La solution : Asciidoctor
- Syntaxe légère et expressive (AsciiDoc)
- Documentation au plus près du code
- Intégration applicative (Maven, Gradle, etc.)
- Options de personnalisation du rendu (HTML, PDF, ePub…)

Faire de la documentation n'est pas forcément une prise de tête.
